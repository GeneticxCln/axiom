# Axiom v0.1.0-alpha.3

> **Release posture:** Axiom is an **alpha compositor prototype with a strong nested development path**.

## Summary

- **Codebase stripped ~2500 lines** — removed DRM backend, effects system, XWayland, clipboard bridge, render bridge, and 11 unused dependencies
- **Rebuilt real GLES rendering** — client `wl_buffer`s imported into `GlesTexture` and drawn via `SolidColorRenderElement`/`TextureRenderElement`. Real pixels on screen.
- **Three long-standing bugs fixed** — decoration button flag inversion, IPC workspace commands not triggering redraw, `WinitEvent::Redraw` handler ignoring OS requests
- **165 tests passing** (zero clippy warnings across all targets, clean `cargo fmt --check`)

## Recommended evaluation path

```bash
cargo build && xvfb-run -a bash ./scripts/nested_smoke_test.sh ./target/debug/axiom
```

Interactive evaluation:

```bash
cargo run -- --windowed --debug
```

Then connect `weston-terminal` against the advertised socket.

## Highlights

### Simplification / cleanup
- **DRM backend removed** — winit-only compositor. The `--backend` flag, `BackendKind::Drm`, `initialize_drm()`, DRM hotplug, libseat session, and all DRM-specific match arms are deleted.
- **Effects system removed** — `EffectsConfig`, `EffectsControl`, `no_effects`/`effects_demo`/`demo` CLI flags, and all WGSL shader code deleted.
- **XWayland removed** — `xwayland/` directory, `xwm.rs`, `XWaylandConfig`, and all xwayland protocol handling deleted.
- **Render bridge removed** — direct `GlesRenderer` surface import, no WGPU intermediate pipeline. Deleted `clipboard_bridge.rs` and `render_bridge.rs`.
- **Config simplified** — removed `effects`, `xwayland`, `backend` config sections and their validation/merge/test code.
- **11 unused dependencies dropped** — `libseat`, `calloop`, `drm`, `drm-fourcc`, `gbm`, `input`, `udev`, and others removed from `Cargo.toml`.
- **Dead code scrubbed** — 8 pre-existing `dead_code` warnings cleaned, unused fields/functions deleted.

### Bug fixes
- **Decoration button flag inversion** (`backend/mod.rs:1595`) — caller no longer overwrites `decoration_consumed_press` flag that `handle_decoration_button()` already set. Window decorations now respond correctly on first click.
- **IPC workspace commands no longer silent** — all 9 IPC workspace methods (`ActivateWorkspace`, `SendToWorkspace`, etc.) now call `request_redraw()` after state change. Previously, workspace switching via IPC produced no visible result until the next redraw trigger.
- **`WinitEvent::Redraw` handler fixed** — OS-level redraw requests are now honored instead of being silently dropped.

### Rendering
- **Real client pixels** — `render()` binds the winit GLES backend, imports each client `wl_buffer` into a `GlesTexture`, draws it (plus backdrop and SSD titlebars/buttons) via `SolidColorRenderElement`/`TextureRenderElement`, then submits.

## Known limitations

- Single output only (hardcoded 1920×1080 virtual size). Multi-output infrastructure exists but is not wired.
- Fractional scale is advertised to clients but sourced from the workspace tape, not the output.
- Render performance still does full-frame redraws (no damage tracking).
- Server-initiated drag-and-drop is not wired — no trigger mechanism exists.
- Touch gesture support limited to basic tap-to-click.

## Build / packaging notes

- No new dependencies added. 11 removed: `libseat`, `calloop`, `drm`, `drm-fourcc`, `gbm`, `input`, `udev`, `libloading`, `gl`, `mockall`.
- The `--windowed` CLI flag is retained but now redundant (the compositor always uses winit).
- The `--debug` CLI flag is retained for convenience.
- Session wrapper (`axiom-session`) and packaging assets (Arch PKGBUILD, Flatpak manifest) updated to reflect winit-only posture.

## Upgrade notes

Config changes from v0.1.0-alpha.2:
- The `effects` config section is removed (was a no-op since alpha.1).
- The `xwayland` config section is removed.
- The `general.backend` config field is removed.
- All other existing config fields remain backward compatible.

## Full notes

Axiom is a winit-only Wayland compositor (GLES rendering) with niri-inspired scrollable workspaces. This alpha is a major cleanup release that strips ~2500 lines of dead code, removes all vestigial subsystems (DRM, effects, XWayland, clipboard bridge, render bridge), rebuilds GLES rendering from scratch with real client texture import, and fixes three long-standing rendering/input bugs.
