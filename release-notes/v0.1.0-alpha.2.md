# Axiom v0.1.0-alpha.2

> **Release posture:** Axiom is an **alpha compositor prototype with a strong nested development path**.

## Summary

- **Server-side decorations enabled** — visible titlebars, buttons, and title text via WGPU solid-color pipeline
- **Config-driven runtime features** — `general.debug`, `general.vsync`, and `input.mouse_accel` now wired to live behavior
- **233 tests passing** (187 unit + 2 bin + 44 integration) with zero clippy warnings and clean security audits
- Release build regression fixed — `create_solid_pipeline` and `anyhow::Context` import no longer gated behind debug assertions

## Recommended evaluation path

The recommended way to evaluate Axiom remains the nested/windowed path:

```bash
cargo run -- --windowed --debug
```

Real nested smoke path:

```bash
xvfb-run -a bash ./scripts/nested_smoke_test.sh ./target/debug/axiom
```

## Highlights

### Compositor/runtime
- **Server-side decorations live** — titlebar backgrounds, close/maximize/minimize buttons, and title text now render via WGPU. CSD-first policy removed; the compositor negotiates ServerSide with clients by default.
- **general.debug config** — set `general.debug = true` in `axiom.toml` to enable debug-level logging at runtime without restarting
- **general.vsync config** — controls WGPU present mode: `true` (default) selects `Fifo`, `false` selects `Immediate` (tearing, lower latency)
- **input.mouse_accel config** — `[-1.0, 10.0]` range applied to libinput pointer devices on hotplug (clamped to libinput's `[-1.0, 1.0]` with debug log when clamping occurs)
- Release builds now compile cleanly — `create_solid_pipeline` and `anyhow::Context` import fixed

### Testing / CI
- **290 tests** across all suites pass cleanly — unit, integration, IPC fuzz, config property, backend
- Zero clippy warnings with `-D warnings`
- `cargo fmt -- --check` clean
- `cargo-audit` and `cargo-deny` both pass (RUSTSEC-2026-0192 ttf-parser documented as ignored)
- Benchmarks compile and link in release mode

### Bug fixes
- `deny.toml` syntax error (extra `]` bracket) — cargo-deny now parses cleanly

- `anyhow::Context` import gated behind debug assertions — `.context()` calls failed in release

## Known limitations

- The standalone DRM/KMS path is still an early/in-progress target; root/SUID or seatd required for device access
- Title text rendering depends on system font availability — fails gracefully with empty titlebars when fonts are missing
- `general.vsync` only affects newly created surfaces; existing surfaces require a resize/`add_output` to pick up the change
- XWayland support is improving, but broader application compatibility still needs more validation
- No session management via logind/seatd — DRM opens `/dev/input/event*` directly
- No XWayland sandboxing (Landlock, seccomp, or namespace isolation)

Reference:
- `docs/user/LIMITATIONS.md`

## Build / packaging notes

- System dependencies: wayland, libxkbcommon, mesa, libinput, systemd-libs, libudev, vulkan-icd-loader
- Session wrapper at `/usr/bin/axiom-session` discovers config in priority order: `$XDG_CONFIG_HOME` → `~/.config` → `/etc/axiom/axiom.toml` → built-in defaults
- Desktop entries installed to `/usr/share/applications/axiom.desktop` and `/usr/share/wayland-sessions/axiom.desktop`

## Upgrade notes

No special upgrade steps required for this alpha. Config changes from v0.1.0-alpha.1:
- `input.mouse_accel` now applied to libinput devices (previously accepted but unused)
- `general.vsync` now applied to WGPU present mode (previously accepted but unused)
- `general.debug` now controls runtime log level (new field)

All existing config files remain backward compatible.

## Full notes

Axiom is a hybrid Wayland compositor combining scrollable tiled workspaces with GPU-accelerated visual effects. This alpha release wires up previously dormant config fields, enables live server-side decoration rendering, fixes release build regressions, and brings all test suites to zero-failure with clean static analysis.
