## Known limitations

- The standalone DRM/KMS path is still an early/in-progress target; root/SUID or seatd required for device access
- Visible server-side decorations are not fully integrated into the live output path; current live policy remains **CSD-first**
- XWayland support is improving, but broader application compatibility still needs more validation
- No session management via logind/seatd â€” DRM opens `/dev/input/event*` directly
- No XWayland sandboxing (Landlock, seccomp, or namespace isolation)

## Full notes

Axiom is a hybrid Wayland compositor combining scrollable tiled workspaces with GPU-accelerated visual effects. This alpha release establishes the core architecture: WGPU rendering pipeline, config-driven effects, multi-monitor support, IPC subsystem, and CI automation. The nested development path is the primary evaluation target; DRM/KMS mode is functional but requires manual setup.
